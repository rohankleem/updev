## Form-data flow

Verified via inspecting the form's serialized output:

```
serializedKeys = [
  "INPUT/hidden/event_name[]",
  "INPUT/hidden/event_name[]"
]
```

Only the hidden field carries `name="event_name[]"`. Select and custom-input have no name attribute (UI-only). The save handler will only see hidden field values, which are kept in sync by the change handlers. Save round-trip is byte-identical to the previous (text input only) implementation — no save handler changes required.
